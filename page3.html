Concepts fondamentaux de Docker

1. Images Docker

Une image est un modèle immuable contenant tout ce dont une application a besoin pour s’exécuter : code, bibliothèques et configuration. On peut imaginer une image comme une “photo” de l’environnement de l’application.


# Lister les images disponibles
docker images

# Télécharger une image depuis Docker Hub
docker pull nginx

2. Conteneurs Docker

Un conteneur est une instance en cours d’exécution d’une image. C’est l’environnement isolé où votre application tourne.


# Lancer un conteneur
docker run -d -p 8080:80 nginx

# Lister les conteneurs en cours
docker ps

# Arrêter un conteneur
docker stop

3. Dockerfile

Le Dockerfile est un fichier texte décrivant étape par étape comment construire une image Docker.


# Exemple Dockerfile simple
FROM python:3.11
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "app.py"]

4. Volumes Docker

Les volumes permettent de stocker des données en dehors des conteneurs, pour qu’elles persistent même si le conteneur est supprimé.


# Créer un volume
docker volume create mon-volume

# Lancer un conteneur avec un volume
docker run -d -v mon-volume:/data nginx

5. Commandes de base

  • docker build : construire une image depuis un Dockerfile
  • docker run : lancer un conteneur
  • docker ps : lister les conteneurs actifs
  • docker stop : arrêter un conteneur
  • docker rm : supprimer un conteneur
  • docker rmi : supprimer une image
